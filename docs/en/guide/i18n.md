# Internationalization

Built-in locales: `zh-CN`, `en-US`.

## Basic Usage

```vue
<YanivEditor locale="zh-CN" />
<YanivEditor locale="en-US" />
<YanivInlineEditor locale="en-US" v-model:content="html" />
```

`normalizeLocaleCode` mappings:

- `zh-TW` / `zh-HK` → `zh-CN`
- `en` → `en-US`
- Other `zh*` → `zh-CN`, `en*` → `en-US`

## Multi-Instance Isolation

Multiple editors on the same page can use different `locale` values independently. Demo: `/multi-instance`.

## Custom Messages

```ts
import { createI18n, loadLocale, ensureLocalesLoaded } from "@yanivjs/yaniv-editor";

const i18n = createI18n({
  locale: "zh-CN",
  messages: {
    "zh-CN": { editor: { placeholder: "Custom placeholder…" } },
  },
});
```

Extensions read copy via a static `ctx.locale` snapshot within the session; chrome components read instance locale via `inject`. **Do not** call global `t()` directly inside components.

## locale and session

Changing `locale` updates `sessionKey` and triggers session rebuild.
